Running Qwen3 30B Locally: A Practical Guide for AMD GPUs

How I got a 30-billion parameter model running on a consumer RX 7900 XT with Ollama and ROCm. Setup, benchmarks, and real-world performance for content generation.

Why Local LLMs?

Cloud APIs charge per token. When you're running an automated content pipeline that generates 10+ articles per day, those costs add up fast. At ~$0.03 per 1K output tokens on GPT-4, a single 2000-word article costs roughly $0.40. That's $4/day, $120/month — just for text generation.

A local model running on hardware you already own? $0/month after the electricity bill. And for content generation, the quality difference between Qwen3 30B and GPT-4 is negligible for structured blog content.

Hardware Requirements

The Qwen3 30B MoE (Mixture of Experts) model in Q4_K_M quantization needs approximately 18GB of VRAM. Here's what I'm running:

  • GPU: AMD Radeon RX 7900 XT (20GB VRAM)
  • CPU: AMD Ryzen 9 7900X
  • RAM: 32GB DDR5
  • OS: Windows 11

The 20GB of VRAM on the 7900 XT is the key factor. The model fills most of it (~18GB), leaving just enough headroom for the inference engine overhead.

Setup with Ollama

Ollama auto-detects the RX 7900 XT via ROCm on Windows. No extra GPU driver configuration needed.

# Install Ollama (Windows)
# Download from https://ollama.ai

# Pull the quantized model
ollama pull qwen3:30b-a3b-q4_K_M

# Test it
ollama run qwen3:30b-a3b-q4_K_M "Write a one-paragraph summary of transformer architecture"

Performance Benchmarks

On my RX 7900 XT, the 30B MoE model produces tokens at approximately 20-40 tokens per second. For context, a 2000-word blog post (~2700 tokens) generates in about 70-135 seconds. That's fast enough for automated pipelines where latency isn't critical.

Compared to the smaller 4B model running on my secondary Lubuntu system (i7-3820 + GTX 1060 6GB), the 30B model produces noticeably better structured content, especially for technical writing that requires accurate terminology and logical flow.

Integration with OpenClaw

The real power comes from connecting Ollama to an agent framework. I use OpenClaw as the orchestration layer — it schedules content generation tasks, manages the writing pipeline (drafting → SEO optimization → publishing), and communicates through Discord.

The configuration is straightforward: point OpenClaw at your Ollama endpoint, define your skills (blog writer, SEO optimizer, content repurposer), and set a cron schedule. The machine writes while you sleep.

The best automation is the kind you forget is running — until you check your analytics and see traffic growing from content you didn't manually write.

Tips and Gotchas

  • Keep concurrency at 1 agent / 2 sub-agents with the 30B model. Going higher will cause OOM errors.
  • The 14B model makes a good fallback if the 30B fails to load.
  • Monitor VRAM usage with rocm-smi or Task Manager. If you're consistently above 19GB, drop to the 14B.
  • For the Lubuntu secondary system, stick with the 4B quantization. It fits comfortably in 6GB VRAM.

What's Next

I'm working on adding a hybrid video generation pipeline to the same system. The 7900 XT can run ComfyUI with Stable Diffusion XL for image generation, while cloud APIs handle the image-to-video step. More on that in a future post.

Automated Content Pipelines with OpenClaw and Local AI

Building a zero-cost content factory that writes, optimizes, and publishes blog posts on autopilot using local LLMs and a Discord-controlled agent framework.

The Architecture

The pipeline is simple in concept: an AI agent framework (OpenClaw) orchestrates a local LLM (Qwen3 via Ollama) to write, optimize, and schedule content. You communicate with it through Discord and it runs on a cron schedule.

The five core skills in the pipeline are:

  • Blog Writer — Generates full-length articles from a topic prompt
  • SEO Optimizer — Restructures content for search engines
  • Social Media — Creates platform-specific posts from blog content
  • Newsletter — Drafts weekly email digests
  • Content Repurposer — Turns 1 blog post into 13-19 pieces

Cost Breakdown

This is where it gets interesting. The entire pipeline runs on hardware I already own:

Component          | Monthly Cost
-------------------|-----------
Ollama / Qwen3     | $0 (local)
OpenClaw           | $0 (open source)
Electricity (est.) | $8-12
Domain + hosting   | $10-15
-------------------|-----------
Total              | ~$20/month

Compare that to a cloud-based stack using GPT-4 + Jasper + scheduling tools, which would run $150-300/month for similar output volume.

Scaling on Two Systems

I run two setups. The Windows machine (Ryzen 9 + RX 7900 XT) handles the heavy lifting with the 30B model — 10+ articles per day with parallel execution. The Lubuntu machine (i7-3820 + GTX 1060) runs the smaller 4B model at 3-5 articles per day.

Together, that's roughly 15 articles daily across different niches. The content repurposer then multiplies each article into social posts, newsletter snippets, and thread formats.

The moment you automate a repeated task is the moment you start compounding your time.

Monetization

The content feeds into affiliate marketing sites. Each niche blog is built around a specific topic with relevant affiliate links. The SEO optimizer skill ensures articles are structured for long-tail keywords with proper heading hierarchy, meta descriptions, and internal linking.

It takes 2-3 months for SEO to compound. The posts written today generate traffic six months from now. That's the definition of passive income — upfront effort that pays dividends over time.

The Automation Paradox: When Machines Free Your Time, What Do You Do With It?

I automated most of my income. Then I had to confront the question that productivity culture never answers: what's the point?

The Promise

The pitch of automation is seductive: build systems that work while you sleep, and you'll have unlimited time for the things that matter. I bought this premise completely. I spent months building content pipelines, trading bots, and scheduling systems. And it worked. The machines run. The income comes in.

Then I woke up one Tuesday with nothing urgent to do and realized I had no idea what "the things that matter" actually were.

Productivity Is a Trap Without Purpose

The entire productivity industry is built on an unexamined assumption: that freeing up time is inherently good. But time for what? Nobody asks this question because the answer is uncomfortable. Most of us have been so conditioned by work that we don't know who we are without it.

Seneca noticed this 2,000 years ago. In "On the Shortness of Life," he argued that people don't have short lives — they waste long ones. We fill our days with busywork, meetings, and optimizations that make us feel productive without ever asking what we're producing it all for.

It is not that we have a short time to live, but that we waste a great deal of it. Life is long enough, and a sufficiently generous amount has been given to us for the highest achievements if it were all well invested.

The Automation Paradox

Here's the paradox: the better your automation gets, the less you need to do. The less you need to do, the more you're forced to confront the question of what you actually want. And most people have never seriously thought about this.

I found myself doing what most people do when they suddenly have free time — I built more automation. Not because I needed to, but because building was the only thing I knew how to do. The tool had become the goal.

What the Stoics Got Right

The Stoics had a framework for this. They distinguished between things "up to us" (our judgments, values, character) and things "not up to us" (external outcomes, other people, market conditions). The automated systems handle the "not up to us" part. But no amount of automation can handle the internal work.

Marcus Aurelius ran the Roman Empire and still made time each morning to write philosophy in his journal. Not because it was productive, but because examining life was the point of having one.

Where I Landed

I don't have this figured out. But I've started making some changes:

  • I read philosophy for an hour every morning. Not self-help. Actual philosophy — Seneca, Epictetus, Camus, Simone Weil.
  • I write essays that have nothing to do with SEO or monetization.
  • I spend time on problems that can't be optimized — relationships, health, understanding the world.
  • I let some things be inefficient on purpose.

The machines keep running. The income keeps coming. But the question "what's the point?" doesn't have a technical solution. And I think that's actually the most important thing I've learned from building all this automation.

The One-Person Company Is the Future of Business

AI agents, automated pipelines, and zero employees. We're entering an era where one person can run what used to require a team of 50.

The Old Model Is Dying

The traditional business scaling model goes: founder builds thing → hires team → team scales thing → founder manages team. By the time you have 20 employees, you're not building anymore — you're managing people who are building. The founder becomes an HR department with a vision board.

AI changes this equation entirely. Not in the vague "AI will transform everything" way that LinkedIn thought leaders love. In a concrete, measurable way: tasks that used to require hiring someone now cost $0-40/month in API calls or run free on local hardware.

What One Person Can Do in 2026

Here's what a single person with the right automation stack can operate today:

  • Content production: 10-15 optimized blog posts per day across multiple niches, including social media distribution and newsletter generation. Cost: ~$20/month in electricity.
  • Trading operations: Automated copy-trading across prediction markets, monitoring 25+ wallets, executing trades 24/7 with auto-withdrawal. Cost: $0.
  • Video production: Short-form video content from script to final cut, combining local image generation with cloud video APIs. Cost: $0-40/month.
  • Customer service: AI chatbots handle 90% of support queries. Cost: negligible.
  • Website management: AI-generated and optimized landing pages, A/B tested automatically. Cost: hosting fees only.

Five years ago, that list would require a content team, a quant trader, a video editor, a support agent, and a web developer. Today it requires one person and a good GPU.

The Economic Implications

This isn't just a lifestyle optimization. It's a structural shift in how value is created and captured. When one person can do the work of a small company, the economics change dramatically:

  • Near-zero marginal costs. Every additional article, video, or trade costs almost nothing. Scaling doesn't require proportional resource increases.
  • 100% profit capture. No salaries, no office, no HR overhead. Revenue minus hosting and electricity equals profit.
  • Infinite pivotability. Want to enter a new niche? Deploy a new skill to your agent framework. No hiring, no training, no onboarding.
The future of business isn't fewer employees. It's zero employees. The founder IS the company, amplified by machines.

What This Means for Society

If one person can produce the output of fifty, what happens to the other forty-nine? This is the uncomfortable question that futurists love to hand-wave with "new jobs we can't imagine yet."

I don't have the answer. But I know that pretending the shift isn't happening doesn't help. The people who understand these tools will capture disproportionate value. The people who don't will be competing for fewer remaining positions. The transition period is going to be messy.

The best advice I can give: learn to work with AI systems now, while the knowledge gap still matters. In five years, it'll be table stakes.

Things I Changed My Mind About in 2025

Crypto, remote work, social media, morning routines, and a few other positions I've completely reversed on.

I try to update my beliefs when the evidence changes. Here's what shifted for me over the past year.

Crypto: From Skeptic to Cautious Participant

I spent years dismissing crypto as speculation dressed up as revolution. Then I started building Polymarket bots and realized prediction markets are genuinely useful — they aggregate information better than polls, pundits, or news organizations.

I still think 95% of the crypto space is noise. But the 5% that works — stablecoins for cross-border payments, prediction markets for information aggregation, DeFi for programmable finance — actually solves real problems. My mistake was writing off the entire category because most of it is junk.

Remote Work: It's Not For Everyone

I was a zealous remote work advocate in 2020-2023. Work from anywhere, set your own hours, optimize your environment. The dream.

Then I watched friends struggle with isolation, saw junior developers stall without in-person mentorship, and noticed my own productivity fluctuate wildly without external structure. Remote work is great for self-directed senior people building defined projects. It's terrible for collaborative ideation, cultural transmission, and people who need social motivation.

The answer isn't "remote vs. office." It's "what kind of work are you doing, and what kind of person are you?"

Social Media: Quit Most, Double Down on One

I used to be on everything — Twitter, Reddit, LinkedIn, Instagram. The constant context-switching was killing my focus. In early 2025, I deleted everything except one platform where my audience actually lives.

The result: better content on one channel, more time for deep work, and zero FOMO. The "be everywhere" advice is wrong. Be where your people are, and ignore the rest.

Morning Routines: Overrated

I spent six months doing the full Huberman/Atomic Habits morning routine — cold shower, sunlight, meditation, journaling, workout, all before 8am. It was exhausting and I dreaded mornings.

Now I wake up, make coffee, and start working on whatever I'm most excited about. Some days that's at 6am, some days it's at 10am. Energy management beats time management. Your best work happens when you're genuinely engaged, not when an arbitrary schedule says you should be productive.

AI Hype: I Was Too Dismissive

When ChatGPT launched, I thought it was a clever demo that would plateau quickly. I was dead wrong. The improvement curve from GPT-3.5 to where we are now is staggering. Local models on consumer hardware can now do 80% of what cloud APIs did two years ago.

My updated position: AI will be the most transformative technology of our lifetime, but the transformation will be slower, messier, and distributed differently than the hype cycle suggests. It won't replace most jobs overnight. It will slowly change what every job looks like.

The Meta-Lesson

The common thread: I was holding positions based on vibes and identity rather than evidence. "Crypto person" vs. "anti-crypto person." "Remote work advocate" vs. "office advocate." These are tribal identities, not analytical frameworks.

The best thinkers I know hold positions loosely and update frequently. I'm trying to do more of that.

I Built a Gamified Budgeting App Because Spreadsheets Weren't Cutting It

Streaks, XP, badges, and compound growth projections — how I turned personal finance into a game I actually want to play.

The Problem

I track my spending with Rocket Money. It's great at telling me where my money goes. The problem is that knowing where your money goes and actually changing your behavior are two completely different things.

Every month I'd export the CSV, look at the damage, feel bad for 20 minutes, and then continue exactly as before. The data was there. The motivation wasn't.

Why Gamification Actually Works

I've been thinking about habit loops since reading Atomic Habits a few years back. The core insight: behavior change sticks when there's an immediate reward, not a distant one. Saving money has a delayed reward (retirement, emergency fund, freedom). Spending money has an immediate reward (dopamine, stuff, convenience).

Gamification closes that gap. Instead of waiting 30 years to feel the benefit of saving, you get XP, level up, earn badges, and maintain streaks right now. It's the same psychological trick that makes Duolingo addictive — except instead of learning Spanish, you're building wealth.

What I Built

BudgetBoss is a mobile-friendly web app that imports your Rocket Money CSV exports and turns your financial data into a game. Here's what it does:

  • Dashboard with spending breakdown — Visual charts showing where every dollar goes, broken down by category. No more guessing.
  • Streak tracking — Stay under budget for consecutive days and watch your streak grow. Miss a day and it resets. Simple, brutal, effective.
  • XP and leveling system — Earn experience points for staying under budget, hitting savings goals, and maintaining streaks. Level up from Budget Beginner to Financial Boss.
  • Badge collection — Unlock achievements like "First Week Streak," "Savings Starter," and "Investment Initiate." It sounds silly until you realize you're checking the app daily to maintain your progress.
  • Savings goals with visual progress — Set targets (emergency fund, vacation, new GPU) and watch the progress bars fill up over time.
  • Compound growth projections — An investment calculator that shows what your savings become over 10, 20, 30 years at different return rates. Seeing "$500/month becomes $1.1M in 30 years" is more motivating than any budgeting advice.
  • Spending alerts — Category-level warnings when you're approaching your budget limits.

The Tech Stack

I wanted this to work on my phone without installing anything, so I built it as a responsive web app:

Frontend:  React + Tailwind CSS + shadcn/ui
Backend:   Express.js (serves the app)
Data:      CSV import (Rocket Money export format)
Storage:   Local browser storage (no cloud, no accounts)
Hosting:   Any static host or local server

The entire thing runs client-side after the initial load. Your financial data never leaves your device. No accounts, no cloud sync, no privacy concerns. Just you and your numbers.

CSV Import Flow

The workflow is dead simple:

  1. Open Rocket Money, go to Transactions, export as CSV
  2. Open BudgetBoss, click Import, select the file
  3. The app parses your transactions, categorizes spending, and populates the dashboard

The parser handles Rocket Money's export format natively — date, description, category, and amount columns. It auto-maps categories to budget buckets (Food & Drink, Shopping, Transportation, etc.) and flags anything it can't classify for manual review.

Does It Work?

It's been a week. I've checked the app every single day, which is more than I can say for any budgeting tool I've tried. The streak counter is surprisingly motivating — I caught myself at the grocery store thinking "if I put this back, I keep my streak" and that's exactly the kind of in-the-moment decision-making that spreadsheets never triggered.

The compound growth chart is the other secret weapon. When you can see that the $200 you didn't spend this month becomes $4,800 in 20 years at 7% returns, the math of delayed gratification becomes visceral instead of abstract.

The best financial tool isn't the most sophisticated one. It's the one you actually open every day.

What's Next

A few things I want to add:

  • Automated CSV sync — Instead of manual exports, set up a scheduled pipeline to pull data automatically.
  • Social accountability — Optional sharing of streaks and levels with a friend or partner for mutual motivation.
  • Custom budget rules — AI-generated budget suggestions based on spending patterns and income.
  • PWA support — Install it as a home screen app for true native feel.

The code is a single-page React app. If you want to try it yourself or fork it for your own use, the whole thing runs locally with no external dependencies. Your data stays on your device.
